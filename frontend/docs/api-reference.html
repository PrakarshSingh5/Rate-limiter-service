API Reference

Comprehensive documentation for the Rate Limiter REST API.

Check Limits

POST /v1/check

Verifies if a request is allowed based on current usage and limits.

Request Body

{
  "key": "string",          // [Required] Unique identifier (e.g., user:123, ip:1.2.3.4)
  "endpoint": "string",     // [Optional] The resource being accessed
  "ruleId": "uuid",         // [Optional] ID of a specific rule to apply
  "algorithm": "string",    // [Optional] 'token_bucket' | 'fixed_window' | 'sliding_window'
  "limit": "number",        // [Optional] Override usage limit
  "window": "number"        // [Optional] Override time window in seconds
}

Success Response (200 OK) 

{
  "allowed": true,
  "limit": 100,
  "remaining": 99,
  "resetAt": "2024-03-20T10:00:00.000Z",
  "currentUsage": 1,
  "retryAfter": 0
}

Rate Limit Exceeded (429 Too Many Requests) 

{
  "allowed": false,
  "limit": 100,
  "remaining": 0,
  "resetAt": "2024-03-20T10:00:00.000Z",
  "error": "Rate limit exceeded",
  "retryAfter": 45  // Seconds until limit resets
}

Manage Rules

Endpoints for creating and managing persistent rate limit rules.

POST /v1/rules

Create a new rate limit rule.

Request Body

{
  "name": "Global API Limit",
  "algorithm": "token_bucket",
  "limit": 1000,
  "window": 60,
  "keys": [
    { "type": "user", "value": "*" }
  ],
  "webhookUrl": "https://api.myapp.com/webhooks/rate-limit",
  "enabled": true
}
GET /v1/rules

retrieve a list of all configured rules.

GET /v1/rules/:id

Get details of a specific rule by its UUID.

PUT /v1/rules/:id

Update an existing rule.

Request Body

{
  "limit": 2000,
  "enabled": false
}
DELETE /v1/rules/:id

Delete a rule permanently.
