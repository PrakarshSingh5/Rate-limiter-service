Getting Started

Follow this guide to set up the Rate Limiter Service in your environment.

1. Prerequisites

Before you begin, ensure you have the following installed:

  • Node.js (v18 or higher)
  • Redis (v6 or higher) - Used for distributed state management
  • npm or yarn

2. Installation

Clone the repository and install the dependencies:

# Clone the repository
git clone https://github.com/yourusername/rate-limiter-service.git

# Navigate to the project directory
cd rate-limiter-service

# Install dependencies
npm install

3. Configuration

Create a .env file in the root directory. You can start by copying the example file:

cp .env.example .env

Edit the .env file to match your environment. Key settings include:

# Server Configuration
NODE_ENV=development
REST_PORT=3000
GRPC_PORT=50051

# Redis Configuration
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0

View full configuration guide →

4. Running the Service

You can run the service in development mode or build it for production.

Development Mode

npm run dev

Production Build

npm run build
npm start

Using Docker

# Build and start with Docker Compose
npm run docker:up

# Stop the service
npm run docker:down

5. Verify Installation

Once the service is running, you can test the health endpoint to ensure everything is working correctly.

curl http://localhost:3000/health

# Expected output:
# {"status":"ok","uptime":123.45}
