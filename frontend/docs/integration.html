Integration Guide

Learn how to integrate the Rate Limiter Service into your application using REST or gRPC.

1. Integration Workflow

The typical integration flow follows these steps:

  1. Define Rules: Create rate limit rules (e.g., "100 requests per minute per user") via the Rules API.
  2. Intercept Requests: In your application middleware or gateway, intercept incoming requests.
  3. Check Limit: Call the Rate Limiter Service's /v1/check endpoint.
  4. Enforce Decision:
    • If allowed: true, proceed with the request.
    • If allowed: false, reject the request with HTTP 429.

2. Node.js (Express) Integration

Here is a robust example of an Express.js middleware that integrates with the Rate Limiter Service.

const express = require('express');
const axios = require('axios');
const app = express();

// Configuration
const RATE_LIMITER_URL = process.env.RATE_LIMITER_URL || 'http://localhost:3000';

// Middleware Function
const rateLimitMiddleware = (ruleId = null) => {
    return async (req, res, next) => {
        try {
            // Identify the user (e.g., from auth token, IP, or API key)
            const userId = req.user?.id || req.ip;
            
            // Construct the check request
            const payload = {
                key: `user:${userId}`,
                endpoint: req.baseUrl + req.path,
                // Optional: Provide a specific rule ID if you want to enforce a pre-configured rule
                // ruleId: ruleId 
            };

            // Call Rate Limiter Service
            const response = await axios.post(`${RATE_LIMITER_URL}/v1/check`, payload);
            const { allowed, limit, remaining, resetAt, retryAfter } = response.data;

            // Set standard RateLimit headers
            res.set({
                'X-RateLimit-Limit': limit,
                'X-RateLimit-Remaining': remaining,
                'X-RateLimit-Reset': resetAt
            });

            if (!allowed) {
                // Reject request
                res.status(429).json({
                    error: 'Too Many Requests',
                    message: 'Rate limit exceeded. Please try again later.',
                    retryAfter: retryAfter
                });
                return;
            }

            // Allow request
            next();
        } catch (error) {
            console.error('Rate Limiter Error:', error.message);
            // Fail open strategy: Allow request if rate limiter is down
            next();
        }
    };
};

// Apply globally or to specific routes
app.use(rateLimitMiddleware());

app.get('/api/resource', (req, res) => {
    res.json({ data: 'This is a protected resource' });
});

app.listen(8080, () => console.log('App running on port 8080'));

3. Python (FastAPI) Integration

Example dependency for FastAPI to check rate limits.

from fastapi import FastAPI, Depends, HTTPException, Request, Response
import httpx
import os

app = FastAPI()
RATE_LIMITER_URL = os.getenv("RATE_LIMITER_URL", "http://localhost:3000")

async def check_rate_limit(request: Request, response: Response):
    # Identify user
    client_ip = request.client.host
    user_key = f"ip:{client_ip}"
    
    async with httpx.AsyncClient() as client:
        try:
            r = await client.post(f"{RATE_LIMITER_URL}/v1/check", json={
                "key": user_key,
                "endpoint": request.url.path
            })
            r.raise_for_status()
            data = r.json()
            
            # Set headers
            response.headers["X-RateLimit-Limit"] = str(data["limit"])
            response.headers["X-RateLimit-Remaining"] = str(data["remaining"])
            response.headers["X-RateLimit-Reset"] = data["resetAt"]
            
            if not data["allowed"]:
                raise HTTPException(
                    status_code=429, 
                    detail={
                        "error": "Rate limit exceeded",
                        "retryAfter": data.get("retryAfter")
                    }
                )
                
        except httpx.RequestError:
            # Optimize: Log error and fail open
            pass

@app.get("/items/")
async def read_items(dependencies=[Depends(check_rate_limit)]):
    return [{"item_id": "Foo"}]
